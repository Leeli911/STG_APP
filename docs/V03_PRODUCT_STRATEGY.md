# v0.3 产品策略与交付计划

## 为什么是 v0.3，而不是 v3

STG 采用语义化版本：`主版本.次版本.修订版本`。

- `0.x` 表示产品仍在验证核心用户、核心需求和训练方法，接口与流程可能继续调整。
- `0.3` 表示 1.0 前的第三次产品迭代，不等于“第三代成熟产品”。
- `v3.0.0` 通常意味着已经发布过稳定的 v1、v2，并发生第三次不兼容的大版本升级；当前项目没有这些事实。

本轮正式版本号为 `v0.3.0-beta.1`。进入 `v1.0.0` 的前提不是页面更多，而是核心训练闭环经过真实用户验证并能稳定交付。

## 产品要解决的任务

首要用户：知道一些表达方法，但在临时汇报、回答主管问题时仍容易铺垫过长、目的不明或重点混在一起的中文职场用户。

用户任务：

> 当我需要临时向主管或协作方说明一件事时，我希望用几分钟练习一个结构动作，从而在没有提示的下一次表达中，也能让对方迅速知道我的判断和请求。

v0.3 不训练“完整沟通能力”，只训练三个可观察的微技能：

1. 明确目的：回答中是否真正出现希望听众知道、决定或行动的内容。
2. 结论先行：核心判断、结果或请求是否出现在第一句话。
3. 两到三点框架：多个理由是否被明确整理为两到三个部分。

这三个技能对应《结构化表达：如何汇报工作、演讲与写作》公开目录中“明确汇报目的”“汇报工作从结论开始”“汇报工作前先确定框架 / 善于讲三点”等主题。产品只依据公开主题重新设计训练，不复制书中原文与案例。

## 最核心卖点

一句话定位：

> 不是帮你把这一次写得更好，而是训练你下一次自己说清楚。

对用户的短版描述：

> 每天五分钟：先无提示回答，只改一个问题，亲自重写，再用新题验证。

### 与直接使用 ChatGPT 的真实差异

| 默认使用通用聊天工具 | STG v0.3 |
| --- | --- |
| 用户要自己写提示词、选择方法和判断标准 | 每次只训练一个已校准的微技能 |
| 常见目标是获得更好的当前答案 | 目标是观察下一次无提示表达行为 |
| 可能直接得到润色稿并复制采用 | 不提供一键采用，必须亲自重写 |
| 反馈范围和尺度随对话变化 | 使用固定规则、同一证据口径和版本号 |
| 是否换题、何时复习由用户主动安排 | 产品强制新题迁移，后续增加次日冷测 |
| 模型结论可能难以复核且涉及内容上传 | 当前免费版在浏览器本地运行，引用原文证据 |

这里不把 ChatGPT 描述成“不能教学”。ChatGPT Study Mode 已支持提问、反思、知识检查和练习题；差异在于 STG 把一个窄技能的训练顺序、内容、规则和掌握证据产品化，用户无需每次设计教练提示词。

### 与 AI 沟通教练产品的真实差异

- Yoodli 已覆盖真实角色扮演、反馈与进度追踪，优势在语音、动态对话和企业训练。
- Orai 已覆盖语速、停顿、填充词、清晰度、单点动作和连续练习，优势在口头表达交付指标。
- STG v0.3 不与它们竞争“更完整的 AI 教练”，而是先占据“中文职场短汇报的结构动作训练”：文本优先、五分钟、零模型成本、规则透明、迁移优先。

当前差异仍是产品切口，不是长期壁垒。未来壁垒只能来自经过标注的中文职场情境库、规则校准数据、真实用户完成与次日迁移数据，以及由这些证据迭代出的课程顺序。

## v0.3 训练闭环

```text
选择一个微技能
→ 面对真实工作情境做无提示冷回答
→ 用一句话写出自己的核心结论
→ 查看一条引用原文的反馈
→ 必须亲自重写
→ 进入未见过的新情境
→ 再次无提示回答与自检
→ 记录冷回答 / 重写 / 迁移三个行为状态
```

不使用百分制“能力分”。当前结果只表示答案是否满足某条透明规则，不代表完整沟通能力，也不证明长期学习效果。

## 两层验收

### 功能验收

- 首页和 `/training-demo` 可匿名打开，移动端与桌面端可用。
- 第一次提交过短时显示明确错误；有效回答必须进入自检，不允许“点击无反应”。
- 冷回答前不泄露方法；反馈后必须手动改写；新题迁移不可跳过。
- 三种规则覆盖 60 个标注样本，反馈证据全部来自原回答。
- 公开训练全过程不发送 App API、模型或数据库请求。
- Lint、Typecheck、Unit、E2E、静态 Demo Build 和 Next Production Build 全部通过。

### 用户价值验收

开发者通过功能测试只能说明“能用”，不能说明“有用”。发版后邀请 5–20 名目标用户，至少完成：

1. 首次使用观察：能否不经解释在两分钟内开始冷回答。
2. 完成证据：首次训练完成率、用时、卡点与放弃位置。
3. 即时迁移：重写后面对新题，目标结构动作是否独立出现。
4. 次日保持：24–72 小时后再给未见题，结构动作是否仍出现。
5. 主观反馈：用户能否具体说出学到了什么、下一次会怎么做；只问“喜不喜欢”不够。
6. 替代性比较：询问用户会选择本产品、直接 ChatGPT 还是不训练，并记录原因。

v0.3 的继续投入门槛建议为：至少 10 个完整训练，首次闭环完成率不低于 60%，至少 5 名用户提供具体反馈，且至少半数完成者在新题中做出目标结构动作。所有数字必须来自真实 Pilot，未采集前不得写成产品成果。

## v0.4 优先级

只有 v0.3 Pilot 证明用户愿意完成闭环后，再按顺序开发：

1. 次日冷测与连续两个不同场景的掌握判定。
2. 三技能进度和弱项复练，不先扩充成大量课程。
3. 将用户真实工作原稿匿名化后转成练习，但必须明确同意。
4. 在规则不能覆盖的高价值问题上试验免费或本地模型；模型是补充，不改变主动练习流程。
5. 语音只在文本结构训练已被验证后进入范围。
