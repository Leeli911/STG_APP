# Product Overview

## 产品定位

Structured Thinking Gym 是一个中文职场结构化表达训练 App。v0.3 解决的不是“生成更漂亮的答案”，而是“知道方法，却不能在临时汇报中自己用出来”。

> 不是帮你把这一次写得更好，而是训练你下一次自己说清楚。

免费公开训练的标准闭环：

```text
无提示冷回答
→ 自己写出核心结论
→ 查看一条引用原文的反馈
→ 亲自重写
→ 面对未见过的新题再次回答
→ 记录冷回答 / 重写 / 迁移的行为状态
```

## v0.3 目标用户与边界

- 用户：需要向主管或协作方做短汇报、临时回答问题的中文职场用户，优先验证职场新人。
- 界面与题目：全中文，不在同一页面混用中英文教学标签。
- 训练范围：明确目的、结论先行、两到三点框架三个微技能。
- 技术边界：公开训练使用浏览器内确定性规则，不调用模型、数据库或 App API。
- 内容边界：参考用户指定书籍的公开主题独立设计，不复制书中原文和案例。
- 非目标：不训练完整沟通能力，不提供语音、演讲仪态、销售角色扮演或开放式 AI 出题。

## 当前能力

### v0.3 已实现

- 三个技能、三道冷题和三道不同的迁移题。
- 作答前隐藏方法和示范答案，保留真实冷回答。
- 用户先写自己的核心结论，再由规则定位最接近的原文证据。
- 每次只反馈观察、听众影响和一个修改动作。
- 原稿未修改时不能提交；产品不提供一键接受 AI 答案。
- 重写后必须完成新题迁移，才能形成一次完成记录。
- 三种可解释状态：本次已做到、已经接近、本次重点修改；不使用百分制能力分。
- 本地保存最近 30 次训练记录；清理浏览器数据即可删除。
- 三个技能各 20 个标注案例用于确定性规则回归。

### 已有但不作为 v0.3 价值前提

- Next.js、Supabase Auth/Postgres/RLS、七天面试课程、Dashboard 与 History。
- Analysis → Coaching AI 管线、Structured Outputs、Repair、事实保护和运行元数据。
- 接受 / 拒绝 / 编辑、重新评分、异步 Job、Webhook 与 Reconciler。

这些能力继续保留，但真实 OpenAI 与 Supabase Staging 验证未完成。v0.3 先用零模型成本验证用户是否愿意完成主动训练闭环，避免用复杂 AI 基础设施掩盖产品需求尚未成立。

## 为什么不是一个聊天框

- 通用聊天工具可以教方法，但用户必须自己设计提示词、练习顺序和评判标准。
- STG 固定“冷答—反馈—主动重写—迁移”的顺序，防止用户直接复制答案。
- STG 记录是否在新情境中做出目标结构动作，而不把模型给出的高分当成能力。
- 规则版本、引用证据和判定边界可复核；当前回答无需上传到第三方模型。

ChatGPT Study Mode、Yoodli 与 Orai 已分别覆盖通用学习、AI 角色扮演和演讲交付反馈。STG 的切口不是“更强 AI”，而是中文职场短汇报的结构动作训练。完整竞品边界见 [v0.3 产品策略](V03_PRODUCT_STRATEGY.md)。

## 验收口径

功能验收只回答“能否稳定完成训练”；产品验收回答“用户是否愿意用、能否迁移”。开发完成后仍必须进行 5–20 人 Pilot，观察首次完成、即时迁移、24–72 小时后冷测和具体访谈反馈。

任何“提升”“留存”“课程有效”等表述都必须来自实际试用数据。单次规则通过只代表当前文本满足该条规则，不代表完整沟通能力。
