# STG 用户价值验证方案

更新日期：2026-07-23

## 1. 需要验证的用户需求

目标用户是在临时汇报中容易铺垫过长、目的不明或重点混在一起的中文职场用户。

核心任务：

> 当我需要临时向主管或协作方说明一件事时，我希望用五分钟练习一个结构动作，使我在没有提示的新情境里也能自己说清楚。

核心价值不是“系统帮我写好了”，而是：无提示回答 → 发现一个问题 → 亲自重写 → 新题仍能使用。

## 2. 当前能证明与不能证明的内容

| 结论 | 状态 | 证据或限制 |
| --- | --- | --- |
| 匿名完成一次完整训练 | 已通过自动化验收 | 冷回答、自检、单点反馈、主动重写、迁移与本地记录均已接通 |
| 免费且不上传回答 | 已通过自动化验收 | 公开训练不调用 App API、数据库或模型服务 |
| 规则结果可复核 | 已通过回归验收 | 180 个冻结样本；Macro-F1 1.000；证据溯源 100% |
| 合成机制值得继续测试 | 已形成诊断信号 | 54 条严格配对轨迹中，STG 组即时迁移高于两个对照；不能外推真人效果 |
| 目标用户觉得有用 | 尚未验证 | 需要 5–20 人真实试用 |
| 产品能安排次日冷测 | 已通过自动化验收 | 到期检测、冷测流程与本地状态已接通 |
| 真人次日仍能自己使用 | 尚未验证 | 需要 14 天开发者自测和后续真实用户纵向数据 |
| 完整表达能力提升 | 不可宣称 | 三条文本规则不能代表完整沟通能力或因果效果 |

## 3. 首轮试用任务

先邀请 5–8 名目标用户，通过门槛后扩至 20 人：

1. 不进行产品讲解，让用户从首页开始第一个五分钟训练。
2. 记录首次输入、提交、查看反馈、重写、迁移与完成的时间和卡点。
3. 让用户解释系统指出的问题，并说明自己为什么这样重写。
4. 让用户完成未见过的迁移题，不展示方法提示。
5. 24–72 小时后发送同技能的新情境，观察是否仍做出目标动作。
6. 询问用户会选择 STG、直接使用 ChatGPT 还是不训练，并追问具体原因。

## 4. 继续投入门槛

- 至少 10 次完整训练，首次闭环完成率不低于 60%。
- 至少 5 份可追溯的结构化反馈，而不是“不错”“界面好看”。
- 至少半数完成者在即时迁移题中独立做出目标结构动作。
- 与输入无关的反馈和虚构引用均为 0。
- 用户能说出一个真实的再次使用场景；如果只能说“可能会用”，不计为强需求证据。

这些是发版后的决策门槛，不是当前已取得的结果。

## 5. 访谈问题

- 最近一次你向主管说明事情却没有说清楚，发生在什么场景？
- 你刚才不看方法时，最难的是哪一步？
- 哪条原文证据让你理解了问题？哪条判断不准确？
- 新题回答时，你主动想起了什么？
- 如果直接用 ChatGPT，你会怎么做？这个训练比它多了什么、又少了什么？
- 未来一周你会在什么具体时刻再次打开？

## 6. 决策规则

- 完成率低：先修交互和题目，不扩课程。
- 完成率高但迁移失败：反馈能帮改稿，尚不能训练能力；加强提取练习和间隔复习。
- 即时迁移成功但次日失败：增加间隔冷测，不宣称掌握。
- 用户更愿意直接用 ChatGPT：找出速度、情境或反馈上的缺口，不用新增功能掩盖定位问题。
- 用户只想复制范文：目标人群或价值主张不匹配；不要恢复一键采用来制造完成率。

## 7. 必须留存的证据

- 匿名漏斗与每阶段用时。
- 冷回答、重写、迁移与延迟冷测的规则状态及规则版本。
- 24–72 小时冷测结果。
- 访谈原始记录与由此形成的产品决策。
- 用户明确同意后才可保存去标识化文本；否则只保留状态和用时。
